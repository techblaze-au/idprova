# Code of Conduct

## Our pledge

The IDProva project exists to build trustworthy identity infrastructure for AI agents. The community we build around it should reflect that goal. Everyone who participates — contributors, maintainers, and users — agrees to make the project a respectful, welcoming place to do technical work, regardless of background, identity, or experience level.

## Our standards

We expect everyone in the project to:

- Be respectful and constructive in all interactions, including code reviews, issues, and discussions.
- Assume good intent. Ask clarifying questions before assuming the worst.
- Give credit where it is due. Cite sources, acknowledge prior work, and link to upstream when borrowing.
- Disagree on the technical merits, not on the person. Keep critique focused on the work.
- Make the project easier for the next contributor: explain decisions, document trade-offs, and link issues to PRs.

We do not tolerate:

- Personal attacks, insults, or sustained disruption of the project's communication channels.
- Posting another person's private information without their explicit permission.
- Sexualised language, imagery, or unwelcome advances toward any participant.
- Conduct that a reasonable person would consider inappropriate in a professional setting.

This list is illustrative, not exhaustive. Maintainers may, at their discretion, treat other behaviour as a violation when it conflicts with the spirit of the project.

## Scope

This Code of Conduct applies in every space the project operates: GitHub repositories under the `techblaze-au` organisation, issues and pull requests, project mailing lists, official chat channels (when established), public events organised by the project, and any other forum where someone is acting on behalf of the project.

## Reporting

If you observe or experience behaviour that violates this Code, contact the maintainers at:

- **Email:** [hello@techblaze.com.au](mailto:hello@techblaze.com.au)

Reports should include: what happened, when and where, who was involved, and any links or screenshots. Reports are kept confidential. The project will not retaliate against anyone who reports in good faith.

For security vulnerabilities, use the channel in [SECURITY.md](SECURITY.md) instead — that path is designed for sensitive technical disclosure.

## Enforcement

Maintainers will review every report. Possible responses include:

- Private feedback or warning to the participant.
- A request to amend or remove specific content (issue, PR comment, etc.).
- Temporary suspension from project communication channels.
- Permanent removal from project participation.

Maintainers will choose the response that fits the severity and context of the violation. Decisions can be appealed by emailing the same address with the subject line "CoC appeal".
